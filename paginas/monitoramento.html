Monitoramento de Conexão com Servidores

A ideia principal do programa é prevenir danos maiores, fazendo um
monitoramento de conexão com todos os servidores internos e sites que são da FUSP.
Se por acaso o servidor não é encontrado ou ter perca de pacotes é disparado
um e-mail de emergência para a equipe de TI.


Clique aqui para ver o código
Tecnologia usada: Python
Bibliotecas: Os, time, smtplib, email.message

Aprofundamento do sistema

O teste funciona com o método ping no prompt de comado, fazendo um teste de conexão
(camada de transporte e de redes) básica com os servidores, todo ping efetuado
é alocado dentro de um arquivo .txt, para poder fazer a análise. Caso o
teste de conexão seja abaixo de 75% de pacotes entregues ou se o
servidor não for encontrado, é disparado um e-mail de urgência para a
equipe de TI. Ele é executado a cada 30 minutos, assim, se ocorrer algum
problema, os usuários serão alertados.
